Git-Kommandos

Command Beschreibung Info
git status Zeigt den Status des Arbeitsverzeichnisses und des Staging-Bereichs an. Die Statusausgabe zeigt keine Informationen bezüglich der committeten Projekt-Historie. Dafür ist das Kommando git log nötig.
git log Zeigt committete Snapshots. Lässt die Projekt-Historie auflisten, sie filtern und nach spezifischen Änderungen suchen. Wenn die Ausgabe einen Bildschirm übersteigt, kann mit der Leertaste gescrollt und mit q beendet werden.
Lässt sich erweitern um weitere Anweisungen, um beispielsweise die Ansicht auf bestimmte Zeiträume zu limitieren oder gezielte Infos zu filtern. Siehe seibert-media.de
git init Mit dem Befehl wird ein neues Git-Repository erstellt. Mehr zu dem Thema unter atlassian.de
Abfolge um Änderungen vom lokalen Repository ins Git-Repository zu laden
git add * Änderungen werden vorgeschlagen. Durch den Stern werden alle Dateien im Verzeichnis hochgeladen. Statt des Sternchens können auch gezielt Dateien genannt werden, die hochgeladen werden sollen.
git commit -m "Nachricht" Änderungen werden bestätigt. In der Nachricht dann eine verständliche Erklärungen für die anderen Entwickler*innen hinterlassen, welche Änderungen in dieser Version durchgeführt wurden.
git push Änderungen werden zu GitHub hochgeladen.
Hier endet die Abfolge
git branch Übersicht: Welche Branches haben wir und in welchem Befinde ich mich?
git branch branchname Erstellt einen neuen Branch. Bei branchname immer den gewünschten (neuen) Branch einfügen. Der neue Branch muss dann noch gepushed werden (siehe unten).
git checkout -b branchname Erstellt einen neuen Branch und wechselt sofort in diesen Branch.
git push -u origin branchname Lädt den neuen Branch hoch.
git checkout branchname Wechselt zu dem gewünschten Branch.
git branch -d branchname Löscht den Branch im lokalen Repository
git checkout main In der Hauptastamm wechseln
git pull Um den aktuellen Stand des des Masters/der Branch in Visual Studio zu laden. Darauf achten, in welcher Branch man sich befindet.
git merge branchname Um den Arbeitsstand eines Branches in den Master zu implementieren. Um den Hauptstamm zu ändern, muss man sich im Master befinden.
git merge origin/master Um in die eigene Branch die aktuelle Version aus dem Master zu laden. Dafür muss man sich dann auch in der eigenen Branch befinden.

JavaScript-Operatoren

Operator Beschreibung Info
Arithmetische Operatoren (Verwendung: Mathematische Berechnungen)
+ Addition
- Subtraktion
* Multiplikation
** Der Potenzierungsoperator (**) gibt das Ergebnis der Potenzierung des ersten Operanden mit dem zweiten Operanden zurück. Beispiel:
let x = 3;
x **= 2;

Ergibt: 9, weil 3 * 3 berechnet wurde.

Beispiel:
let x = 3;
x **= 3;

Ergibt: 27, weil 3 * 3 * 3 berechnet wurde.
/ Division
% Der Modulo-Operator (%) gibt den Restwert zurück, der übrig bleibt wenn ein Operand durch einen anderen geteilt wird. Beispiel:
console.log(13 % 5);

Ergibt: 3, weil 13/5 = 2 mit einem Restwert von 3
++ Inkrementiert einen Operanden, addiert also eins hinzu und gibt einen Wert zurück. Kann davor und danch eingesetzt werden.
Postfix (z.B. x++): der Operand wird um eins erhöht und der Wert vor dem Inkrementieren zurückgegeben. Prefix (z.B. ++x), der Operand wird um eins erhöht und der Wert nach dem Inkrementieren zurückgegeben. 		Beispiel:
let x = 3;
y = x++;
 Ergibt: y = 3; x = 4

let a = 2;
b = ++a;
Ergibt: a = 3, b = 3
-- Dekrementiert den Operanden um eins, zieht also eins vom Operanden ab und gibt einen Wert zurück. Kann genau wie bei ++ vor oder danach angewendet werden. siehe ++
Zuweisungsoperatoren (Verwendung: JavaScript-Variablen Werte zuweisen)
= Ist gleichbedeutend mit x = y Beispiel: x = y
+= Ist gleichbedeutend mit x = x + y Beispiel: x += y
-= Ist gleichbedeutend mit x = x - y Beispiel: x -= y
*= Ist gleichbedeutend mit x = x * y Beispiel: x *= y
/= Ist gleichbedeutend mit x = x / y Beispiel: x /= y
%= Ist gleichbedeutend mit x = x % y Beispiel: x %= y
**= Ist gleichbedeutend mit x = x ** y Beispiel: x **= y
Vergleichsoperatoren (Verwendung: In logischen Anweisungen, um die Gleichheit oder Differenz zwischen Variablen oder Werten zu bestimmen.)
== gleicher Wert
=== gleicher Wert und gleicher Typ
!= nicht gleicher Wert
!== nicht der gleiche Wert und nicht der gleiche Typ
> größerer Wert
< kleiner Wert
>= größerer oder gleicher Wert
<= kleinerer oder gleicher Wert
? ternary operator
Logische Operatoren (Verwendung: Um logische Verknüpfungen zu formulieren.)
&& Operator steht für UND. Werte müssen die verknüpften Bedingungen erfüllen.
|| Operator steht für ODER. Werte müssen eine der verknüpften Bedingungen erfüllen.
! Operator steht für NICHT. Werte werden umgekehrt und so wird aus true beispielsweise false.
